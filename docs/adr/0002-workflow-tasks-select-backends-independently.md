# Workflow tasks select Backends independently

The orchestrator will own provider-neutral artifact schemas and capability contracts, while every workflow task selects a named Backend through a Run Profile. This allows a single run to mix cloud services and local models, and lets cloud, local English, and local Finnish support be built and tested against the same contracts from v0. Language is profile data rather than duplicated orchestration code; separate adapters are justified only when the underlying models expose materially different interfaces. Each Backend must declare its supported capabilities, and preflight validation will reject an incompatible Run Profile rather than silently substituting another Backend. This requires more contract and conformance-test work initially, but prevents provider behavior from leaking into the workflow and avoids a later architectural rewrite.
