# Resume does not silently rerun invalidated work

At Run creation, the selected profile, task instructions, prompts, and schemas will be frozen into the Run Bundle. Ordinary resume will use those assets and reuse completed stages only when their relevant inputs, configuration, and Backend/model revisions still match. It will never repeat invalidated paid or expensive work silently. The explicit command `rerun RUN_DIR --from STAGE` will show the new estimate and create a parent-linked Run Bundle that carries forward verified upstream artifacts while resolving current or supplied assets from the chosen stage onward; it never mutates the parent. This adds storage and a deliberate step after upgrades, but preserves cost control, provenance, resumability, and the meaning of a reproducible Run Bundle.
