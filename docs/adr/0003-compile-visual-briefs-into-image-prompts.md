# Compile Visual Briefs into Backend-specific image prompts

The visual-planning stage will produce provider-neutral Visual Briefs that refer to reusable Style Profiles. A separately selectable `image_prompt_compile` Structured Text Workflow Task will combine those artifacts with continuity information and a bounded target-Backend descriptor to create a validated Image Request. Its instructions are versioned per image Backend, and the Run persists both the compiler Backend/model and target Image Backend; simple/test renderers may select a deterministic compiler implementation. Generative image Backends are the primary production path, while the deterministic stick-figure Backend remains available as an explicitly selected option and contract-test implementation, never a silent substitution after failure. This separation requires dedicated prompt fixtures and evaluations, but it allows prompting to improve independently for GPT Image, Gemini, FLUX, and future Backends without leaking vendor syntax into scripts or duplicating story-planning logic.
